MADL — Mobile Application Description Language  ·  v0.1
TOP-LEFT  ·  NAVIGATION  — how the user moves
atlasComplete map of all decks, cards, transitions. One per app.
stackNavigator: decks pushed onto history. Back = pop.
tabsetNavigator: N decks as parallel persistent destinations.
drawerNavigator: hidden off-screen, revealed by gesture.
flowNamed linear deck sequence. Declared entry + exit.
entryDefault card shown when deck first opens. WML <go>
exitTarget reached on back / cancel / complete. WML <prev>
IDs: {app}.atlas {app}.nav.stk{n} {app}.nav.tab{n} {app}.nav.drw{n} {app}.flow.{name}
Atlas format:  {deck}: entry:{card}  exit:{deck|card}  nav:{nav}
TOP-RIGHT  ·  INTERACTION  — what the user does
triggerGesture or system event that initiates an action. WML <do>
actionNamed operation bound to a trigger. WML <do> type
transitionMovement from card/deck to card/deck caused by action.
guardCondition that must be true for transition to execute.
Gestures (closed): taplong-pressswipe-leftswipe-rightswipe-upswipe-downpinchexpandscroll-endbacksystem-event
Transitions (closed): pushpopreplacemodal-presentmodal-dismisssheet-upsheet-downfadenone
IDs: {card}.t{n} {card}.t{n}.a
Action format:  target:{card|deck}  guard:{condition}  animation:{type}
BOTTOM-LEFT  ·  ELEMENTS  — what populates cards
elementAny discrete UI component placed in a slot. WML generic
fieldElement accepting user input. WML <input>
labelElement displaying static or bound text. WML <p>
controlElement that triggers an action (button, toggle, selector).
mediaElement displaying image, video, or audio.
listRepeating element rendering a bound collection.
overlayElement rendered above the card surface.
Overlay sub-types (closed): dialogsheettoasttooltipaction-sheet
IDs: {slot}.e{n} {card}.o{n}
Props:  type*  label  bound-to  visible-if  modal:true|false  auto-dismiss
BOTTOM-RIGHT  ·  SERVICES  — what the app talks to
serviceAny external system the app communicates with.
storeLocal persistent data (on-device db, keychain, prefs).
endpointNamed operation exposed by a service.
bindingLink between element and store or endpoint.
hostDevice OS capabilities (camera, GPS, biometrics…).
cloudRemote platform service (auth, storage, push…).
eventAsync message from service that may trigger card change.
Host capabilities (closed): cameragpsbiometricsnotificationscontactsmicrophonenfcbluetooth
IDs: {app}.svc.{name} {app}.store.{name} {svc}.ep.{name} {app}.svc.host.{cap} {app}.svc.cloud.{name} {svc}.evt.{name}
